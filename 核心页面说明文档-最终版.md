### 2.1.1 首页 (Home.vue)

**页面截图：**

**页面介绍：**

首页是整个网站的入口，我在设计时考虑了响应式布局，让它在手机和电脑上都能正常显示。下面介绍一下各个功能模块：

### 2.1.2 视频列表页 (Videos.vue)

**页面截图：**
> [请在此处插入视频列表页截图]

**页面介绍：**

这个页面是整个网站的重点，因为要满足大作业对多媒体控制的要求。我在这里实现了视频和音频的播放功能。

**① 页面标题**

顶部显示"视频教程"，就是简单的标题样式，自己写的。

**② 视频播放区域**

我放了3个教程视频，每个视频都有完整的播放控制功能。视频用的是HTML5原生的`<video>`标签，这个是课堂上学的，也参考了MDN文档。视频文件的路径是我自己配置的。

**③ 视频控制面板**

这是重点功能，每个视频右边都有一排控制按钮：
- 播放/暂停：用`video.play()`和`video.pause()`实现
- 快退/快进：通过改变`video.currentTime`实现，每次5秒
- 播放速度：可以选1倍、1.5倍、2倍速，用`video.playbackRate`控制
- 音量调节：通过`video.volume`实现
- 静音控制：用`video.muted`实现
- 时间显示：实时显示当前时间和总时长

Video API的基本用法是课堂学的，也参考了MDN文档。但是控制按钮的布局和样式比较复杂，用了AI帮忙生成。时间格式化函数（把秒数转成"分:秒"格式）也是AI帮忙写的，因为涉及到数学计算和字符串处理。按钮的点击事件是我自己写的。

**④ 视频信息**

视频下面显示标题、描述、时长、观看次数这些信息，就是简单的文字和图标布局，参考了Element Plus文档。

**⑤ 音频播客区域**

在页面下半部分，我做了一个音频播放器，可以播放音乐，支持切歌和进度控制。

用的是HTML5的`<audio>`标签，这个也是课堂学的。进度条的样式和交互逻辑比较复杂，用了AI帮忙生成。歌曲切换的逻辑是我自己写的，播放控制按钮的样式参考了网上的教程。

**用到的技术：**
- HTML5的Video和Audio标签（课堂学的）
- Video API和Audio API（参考MDN文档）
- CSS3的卡片布局、渐变按钮、进度条样式
- JavaScript的DOM操作、事件处理、时间格式化

## 2.4 响应式设计

所有页面都做了响应式设计，在手机和电脑上都能正常显示。

**断点设置**

按照课程要求，我设置的断点是768px。也就是说，屏幕宽度大于768px显示桌面版，小于768px显示移动版。

**怎么实现的**

主要用了两种方法：

**Element Plus栅格系统**

比如这样写：
```vue
<el-col :xs="24" :md="8">
```
意思是手机上占满整行（24格），电脑上占三分之一（8格）。这个用法参考了Element Plus官方文档。

**CSS媒体查询**

在CSS里这样写：
```css
@media (max-width: 768px) {
  .desktop-menu { display: none; }
  .mobile-menu-icon { display: block; }
}
```
意思是屏幕宽度小于768px时，隐藏桌面菜单，显示移动菜单图标。媒体查询的语法是课堂学的，也参考了MDN文档。

**移动端菜单**

电脑上显示横向导航菜单，手机上显示抽屉式菜单。抽屉菜单用的是Element Plus的组件，参考了官方文档。
